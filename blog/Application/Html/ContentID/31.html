MYSQL添加外键时的注意事项

  • 黄磊
  • MySQL
  • Mysql
  • 2017-03-11 11:26:50

sh_article_cat表

id 主键 自增长


sh_article表

ac_id   关联sh_article_cat的id


CONSTRAINT fk_ar_ac FOREIGN KEY (ac_id) REFERENCES sh_articles_cat(id) ON

DELETE RESTRICT ON UPDATE CASCADE


声明一个外键ac_id取别名为fk_ar_ac，此外键受sh_articles_cat表中的id影响，sh_article_cat禁止删除，修改时sh_article中的ac_id会随之变化


创建外键时，可以指定在删除、更新父表时，对子表进行的相应操作，包括RESTRICT、CASCADE、SET NULL 和 NO ACTION。

RESTRICT和NO ACTION相同，是指限制在子表有关联记录的情况下父表不能更新

CASCADE，是指父表在更新或删除时，更新或者删除子表对应记录

SET NULL ，是指父表在更新或删除时，子表的对应字段被SET NULL



需要注意的是，父表中的id不能是unsigned，需要保持数据类型相同，才可创建成功，

否则提示错误：Cannot add foreign key constraint
